PowerfulSeal configuration schema

The following configuration options document all features available for writing policy files with PowerfulSeal.

Type: object

General configuration of this policy.

Type: number

If set, will execute the given scenarios once, and then exit. Use this option for one-off experiments, rather than a process running continuosly on the cluster.

Value must be greater or equal to 1

Type: number Default: 300

The Seal waits a randomized amount of time before executing the scenarios. This sets the MAX number of seconds that it will wait.

Value must be greater or equal to 0

Type: number Default: 0

The Seal waits a randomized amount of time before executing the scenarios. This sets the MIN number of seconds that it will wait.

Value must be greater or equal to 0

Type: array of object

An array of scenarios manipulating the pods in the cluster. The scenarios are executed one by one in the specified order.

Each item of this array must be:

Type: object

Describe a pod scenario. It's a scenario that can match, filter and action on pods in your kubernetes cluster. You can filter by selectors, namespaces, deployments etc. Then you can filter the pods based on various criteria, like state or IP. Then you can action on each of them, by executing the configured actions. See below for the details of the configuration available.
